관찰 — Spec 강화의 실질 효과

  • §5.1 70%→50% 임계 강화 핵심. v1 idea_5 ∩ idea_7 = 60%는 spec 통과였으나 사실 같은 페인 클러스터 변형. v2에서 같은 상황이 발생했더라면 reject. 4건 reject는 spec이 "동일 페인 클러스터 변형"을 자동 압축하고 있다는 명확한 신호.
  • §5.5는 안전망으로 작동. passed 풀이 자연스럽게 2 domain 보장되어 desync_domain reject 0건. 이는 spec 무용이 아니라 "필요 없을 때 무력화되고 필요할 때 작동"하는 graceful degradation의 정상 동작.
  • 페르소나 다양성이 진짜 분기점. v1과 v2 sweep의 passed 도메인 분포는 동일(mobile_pwa 2 + web_saas 1)이었지만 페르소나는 1 → 3으로 분기. health 토픽에선 "어떻게 구현하느냐(domain)"보다 "누구를 위한 것이냐(persona)"가 다양성의 진짜 측정 단위.
  • avg_pain_signal +19%는 입력 다양성의 부산물. 5개 페르소나 풀에서 페인을 발굴하니 각 페르소나에서 가장 강한 페인 후보를 골라 조합 가능. 단일 페르소나는 신호 강도가 빠르게 plateau.
  • 잔존 이슈: 학생·자영업 페르소나는 passed 진출 못 함. idea_4(StudyRhythm, 학생)와 idea_5(FreelancePulse, 자영업)는 기준 통과했으나 max_passed=3 상한에 밀림. 5 페르소나 입력 → 3 페르소나 출력. health 토픽에선 페르소나 5 → 3 압축은 자연스럽지만, max_passed를 4-5로 늘리는 옵션도 검토 가치 있음.
