# 빌딩 파트너 매칭 플랫폼 — 전략적 기획 분석

---

## 1단계 — 아이디어 디섹션 (Idea Dissection)

### 핵심 가치 제안 (Value Proposition)

**"사이드 프로젝트를 시작하지만 끝내지 못하는 개발자들에게, 서로의 책임감을 레버리지로 완성까지 이끄는 빌딩 파트너 매칭 서비스"**

- **누구의 문제인가?**: 아이디어는 넘치지만 혼자서는 완성에 이르지 못하는 개인 개발자·디자이너·기획자
- **어떤 문제인가?**: 동기 저하, 고독감, 책임감 부재로 인한 프로젝트 포기율 (업계 추산 80~90%)
- **어떻게 해결하는가?**: 유사한 기술 스택·관심사·진행 단계를 가진 빌딩 파트너를 매칭하고, 정기 체크인·마일스톤 시스템으로 상호 책임감을 구조화

### 고객 세그먼트 (얼리어답터)

| 세그먼트 | 특징 | 페인포인트 강도 |
|---------|------|--------------|
| **주니어~미드 개발자 (2~5년차)** | 아이디어 풍부, 실행력 불안정, 커뮤니티 갈증 | ★★★★★ |
| 부트캠프 수료생 | 포트폴리오 필요, 혼자 완성한 경험 없음 | ★★★★☆ |
| 대기업 재직 개발자 (부업 욕구) | 시간 제약, 동료 부재 | ★★★★☆ |

**얼리어답터 결론**: 사이드 프로젝트 경험이 1회 이상 있지만 완성 경험이 없는 **2~5년차 개발자**가 첫 번째 설득 대상.

## 2단계 — 기능 명세 (Feature Sets)

*아이디어의 컨텍스트를 기반으로 합리적인 가정 하에 진행합니다.*

### Core (필수 — 없으면 서비스가 안 됨)

| 기능 | 설명 |
|------|------|
| 프로필 & 스택 등록 | 기술 스택, 경력, 희망 프로젝트 도메인, 주당 가용 시간 |
| 프로젝트 아이디어 등록 | 아이디어 제목·한 줄 설명·현재 진행 단계(아이디어/설계/개발 중) |
| 파트너 매칭 | 필터 기반 탐색 + 매칭 요청·수락 플로우 |
| 마일스톤 설정 | 파트너십 시작 시 공동으로 주간/월간 마일스톤 등록 |
| 정기 체크인 알림 | 주 1회 "이번 주 완료한 것 / 다음 주 목표" 입력 리마인더 |
| 프로젝트 완료 선언 | 완성 시 공개 배지·포트폴리오 등록 |

### Enhancement (부가 — 나중에 가능)

| 기능 | 설명 |
|------|------|
| 활동 타임라인 | 체크인 히스토리 시각화 |
| 공개 빌드 로그 | SNS처럼 진행 상황을 외부에 공유 (빌드 인 퍼블릭) |
| 커뮤니티 피드 | 다른 팀의 진행 상황 구경 → 동기 자극 |
| 파트너 평가 시스템 | 완료 후 상호 리뷰 → 신뢰도 점수 |
| Slack/Discord 연동 | 체크인 알림을 기존 채널로 전달 |

### Scale (확장 — 성장 단계)

| 기능 | 설명 |
|------|------|
| AI 매칭 추천 | 과거 완료율 데이터 기반 궁합 예측 |
| 멘토 매칭 | 시니어 개발자가 팀을 자문하는 유료 티어 |
| 기업 채용 연계 | 완성된 프로젝트 기반 채용 파이프라인 |
| 해커톤 모드 | 기간 한정 집중 완성 챌린지 |
| API 개방 | 부트캠프·교육 기관 대상 화이트라벨 |

## 3단계 — MVP 스코핑 (The Lean MVP)

### MVP 범위

Core 기능 중에서도 **가장 얇은 레이어**만 선택:

```
[MVP = 프로필 등록 + 파트너 탐색/매칭 요청 + 마일스톤 설정 + 주간 체크인 알림]
```

공개 배지나 커뮤니티 피드는 MVP 이후로 미룬다.

### 기술적 실현 가능성

| 항목 | 평가 |
|------|------|
| 팀 규모 가정 | 1~2인 (개발자 1인 + 디자이너 또는 풀스택 1인) |
| 예상 MVP 기간 | **6~10주** (Next.js + Supabase 조합 기준) |
| 주요 기술 난이도 | 낮음 — 매칭 알고리즘은 초기에 단순 필터로도 충분 |
| 가장 복잡한 부분 | 알림 시스템 (이메일/푸시)과 체크인 리마인더 스케줄링 |

### 이 MVP로 검증할 가설

1. **가설 A**: 개발자들이 프로필을 등록하고 파트너를 찾으려 할 만큼 '혼자 완성 못 하는 고통'이 실재한다.
2. **가설 B**: 파트너가 생기면 실제 완성률이 올라간다 (매칭 후 4주 이내 마일스톤 달성률 측정).
3. **가설 C**: 구조화된 체크인(매주 리포트)이 이탈을 줄이는 핵심 메커니즘인지 확인.

## 4단계 — 비판적 시장 검증 (Critical Assessment)

### 페인 포인트 검증 — "정말 돈을 낼 만큼 아픈가?"

솔직하게 말하면, **이 페인포인트는 실재하지만 '지불 의사(WTP)'로 연결되기가 어렵다.**

- 사이드 프로젝트 미완성은 대부분 **"돈 문제"가 아니라 "의지와 시간" 문제**로 인식된다.
- 사용자들은 무료로 GitHub를 팔로우하거나, Discord 스터디 그룹을 만들거나, 트위터에서 "빌드 인 퍼블릭"을 하는 방식으로 이미 **무료 대안**을 쓰고 있다.
- "책임감 파트너"라는 개념은 가치 있지만, **유료 서비스로 포지셔닝하기 전에 네트워크 효과(사용자 수)가 선행**되어야 한다.

### 경쟁 분석

| 경쟁자 | 유형 | 차별점 없는 부분 |
|--------|------|----------------|
| **Indie Hackers** | 커뮤니티 | 빌딩 파트너 찾기, 진행 공유 기능 이미 존재 |
| **Twitter/X "빌드 인 퍼블릭"** | SNS | 공개 책임감 메커니즘 동일 |
| **YC Co-founder Matching** | 공동창업자 매칭 | 진지한 창업 파트너 탐색 |
| **ADPList** | 멘토링 | 1:1 성장 파트너 매칭 |
| **Focusmate** | 가상 근무 파트너 | 실시간 책임감 구조 (더 강한 형태) |
| **Discord/Slack 스터디 그룹** | 무료 대안 | 비구조적이지만 0원 |

**핵심 차별점이 될 수 있는 것**: 기존 경쟁자들은 "매칭"이나 "커뮤니티"는 있지만, **프로젝트 완성에 특화된 마일스톤 추적 + 상호 책임 구조**를 통합한 서비스는 없다. 이것이 유일한 진짜 해자(moat)다.

### 실패 리스크 3가지

**리스크 1 — 닭과 달걀 문제 (가장 치명적)**
파트너 매칭 플랫폼은 사용자가 충분해야 좋은 매칭이 나온다. 초기 200~300명 미만에서는 "내 스택에 맞는 파트너가 없어요"라는 이탈이 반복될 것이다. 이 임계점을 어떻게 돌파할지가 생존 관건이다.

**리스크 2 — 낮은 완성률이 플랫폼 평판을 갉아먹는다**
파트너를 매칭해줬는데도 프로젝트를 완성 못 하면, 사용자는 "서비스 탓"을 한다. 완성률 지표가 낮으면 구전(Word of Mouth)이 오히려 역효과를 낸다.

**리스크 3 — 핵심 기능이 무료 대체재로 충분히 복제 가능**
누군가 Notion 템플릿 + 카카오톡 오픈채팅으로 똑같이 운영할 수 있다. 특히 초기 사용자들은 이미 그렇게 하고 있다. 유료화 정당성을 만들기 전에 복제당할 수 있다.

## 5단계 — 수익화 모델 3가지 (Monetization Strategy)

### 모델 A — 프리미엄 구독 (Freemium → SaaS)

| 항목 | 내용 |
|------|------|
| **구조** | 무료: 매칭 1건 + 기본 체크인 / 유료($9~15/월): 무제한 매칭 + AI 추천 + 멘토 접근 |
| **장점** | 사용자 진입 장벽 낮음, 반복 매출 예측 가능 |
| **단점** | 개발자 커뮤니티는 유료 전환 저항이 강함, 이탈률 높을 수 있음 |
| **적합 단계** | MAU 2,000명 이상 확보 후 |

### 모델 B — 채용 연계 수수료 (Recruiting Pipeline)

| 항목 | 내용 |
|------|------|
| **구조** | 기업이 "완성된 사이드 프로젝트"를 가진 개발자를 채용할 때 성공 수수료 (연봉의 5~10%) 또는 인재 풀 접근 구독 |
| **장점** | 사용자에게 직접 과금 안 해도 됨, B2B 단가 높음 |
| **단점** | 채용 시장 경기에 민감, 규모 없으면 기업 설득 어려움 |
| **적합 단계** | 완성 프로젝트 DB가 500개 이상 축적된 후 |

### 모델 C — 기관·부트캠프 화이트라벨 (B2B SaaS)

| 항목 | 내용 |
|------|------|
| **구조** | 코딩 부트캠프, 대학 CS 학과, 해커톤 주최사에 플랫폼을 화이트라벨로 판매 ($200~500/월 per 기관) |
| **장점** | 수익 단가 높음, 초기 유저 확보 채널로도 활용 가능 |
| **단점** | B2B 영업 사이클 길다, 커스터마이징 리소스 부담 |
| **적합 단계** | MVP 완성 직후 파일럿 1~2곳부터 테스트 가능 |

**추천 순서**: 초기에는 **무료로 네트워크 효과 구축** → 중기에 **채용 연계 수수료**로 현금화 → 장기에 **화이트라벨 + 프리미엄 구독** 병행

## 최종 PRD — 빌딩 파트너 (BuildMate)

```markdown
# BuildMate PRD

## 1. Project Overview
사이드 프로젝트를 시작했지만 완성하지 못하는 개발자들을 위한
빌딩 파트너 매칭 & 책임감 관리 플랫폼.
"혼자 만들다 포기"를 "함께 만들어 완성"으로 전환한다.

## 2. Target Audience
**핵심 페르소나: 이지수 (29세, 백엔드 개발자 3년차)**
- 회사 업무 외 사이드 프로젝트 아이디어 5개 이상 보유
- GitHub에 미완성 레포 10개 이상
- 혼자 작업 시 3주 이상 지속한 경험 없음
- 개발 커뮤니티 활동 원하지만 진입점 모름
- 페인포인트: "같이 할 사람이 없어서 흐지부지됨", "내가 어디까지 했는지 기록이 없음"

## 3. User Stories
- As a 사이드 프로젝트 개발자, I want 나와 비슷한 기술 스택을 가진 파트너를 찾고 싶다,
  so that 혼자 하는 것보다 더 오래 프로젝트를 지속할 수 있다.
- As a 빌딩 파트너, I want 매주 서로의 진행 상황을 체크인하고 싶다,
  so that 상대방에게 민폐가 되기 싫어서라도 작업을 계속하게 된다.
- As a 완성자, I want 완성된 프로젝트를 포트폴리오로 공개하고 싶다,
  so that 채용이나 네트워킹에 활용할 수 있다.
- As a 기업 채용 담당자, I want 실제 완성된 사이드 프로젝트가 있는 개발자를 찾고 싶다,
  so that 실행력 있는 인재를 선별할 수 있다.

## 4. Functional Requirements
| 우선순위 | 기능 | 설명 | 레벨 |
|---------|------|------|------|
| P0 | 프로필 등록 | 기술 스택, 경력, 주간 가용 시간 | Core |
| P0 | 프로젝트 아이디어 등록 | 아이디어 제목·설명·현재 단계 | Core |
| P0 | 파트너 탐색 & 매칭 | 필터 기반 탐색 + 요청/수락 | Core |
| P0 | 마일스톤 설정 | 파트너십 시작 시 공동 목표 설정 | Core |
| P0 | 주간 체크인 리마인더 | 이메일/앱 알림 + 간단한 입력 폼 | Core |
| P1 | 활동 타임라인 | 체크인 히스토리 시각화 | Enhancement |
| P1 | 파트너 신뢰도 점수 | 완료율·체크인 성실도 기반 | Enhancement |
| P1 | 커뮤니티 피드 | 다른 팀 진행 상황 공개 구경 | Enhancement |
| P2 | AI 매칭 추천 | 완료율 데이터 기반 궁합 예측 | Scale |
| P2 | 채용 연계 | 완성 프로젝트 기반 채용 파이프라인 | Scale |
| P2 | 화이트라벨 | 부트캠프·교육기관용 API | Scale |

## 5. Success Metrics (KPI)
- **매칭 전환율**: 프로필 등록 후 파트너 매칭 완료율 > 40% (4주 내)
- **완성률**: 매칭된 팀 중 프로젝트 완성 선언 비율 > 25% (3개월 내)
- **체크인 지속률**: 매칭 후 4주간 주간 체크인 유지율 > 60%
- **NPS (Net Promoter Score)**: > 40 (완성 경험자 대상)
- **리텐션**: 완성 후 새 프로젝트 재등록률 > 35%

## 6. Roadmap
- **Phase 1 (MVP, 0~3개월)**:
  프로필/프로젝트 등록, 파트너 매칭, 마일스톤 설정, 주간 체크인.
  목표: 파트너 매칭 50건, 완성 프로젝트 10건
  검증: 매칭 전환율 & 완성률 가설 확인

- **Phase 2 (Growth, 3~9개월)**:
```
