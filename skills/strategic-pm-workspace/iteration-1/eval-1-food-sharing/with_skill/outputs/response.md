# 전략적 기획 파트너 응답 — 1인 가구 소량 식재료 공동구매 앱

---

## 1단계 — 아이디어 디섹션 (Idea Dissection)

좋은 아이디어입니다. 실제로 아픈 지점을 건드리고 있어요. 바로 정리해보죠.

### 핵심 가치 제안 (Value Proposition)

> **"혼자 사는 사람이 대형마트 대용량 식재료를 낭비 없이 구매할 수 있도록, 같은 동네 이웃과 묶음 구매를 연결해주는 로컬 공동구매 플랫폼"**

| 항목 | 정의 |
|------|------|
| **누구의 문제** | 1인 가구 (특히 20~40대 도시 거주자) |
| **어떤 문제** | 대용량 식재료를 혼자 다 소비하지 못해 버리거나, 소포장 제품은 비싸서 단가 손해 |
| **어떻게 해결** | 동네 반경 내 이웃과 실시간 공동구매 매칭 → 대용량 제품을 나눠 구매, 단가 절감 + 낭비 제거 |

### 고객 세그먼트 — 얼리어답터는 누구인가?

첫 번째로 설득해야 할 얼리어답터는 **"절약에 민감한 도시 1인 가구 직장인"**입니다.

- 연령대: 25~35세
- 거주: 서울·수도권 원룸/오피스텔
- 특징: 요리는 하지만 식재료를 반 이상 버린 경험이 있음, 가성비 앱(당근마켓 등)에 익숙함
- 현재 해결 방법: 소포장 제품 구매(비쌈), 편의점 즉석식품 대체, 아예 포기하고 외식

## 2단계 — 기능 명세 (Feature Sets)

| 레벨 | 기능 | 설명 |
|------|------|------|
| **Core (필수)** | 위치 기반 이웃 매칭 | GPS 반경 설정 (500m~2km), 같은 동네 유저 묶기 |
| **Core** | 공동구매 방 개설/참여 | "파 1단 나눌 분 구해요" 형태의 글 등록 및 참여 |
| **Core** | 결제 및 정산 | 참여자 인원 확정 시 자동 1/N 결제 분할 |
| **Core** | 수령 약속 (픽업 조율) | 시간·장소 합의 채팅 (간단한 1:N 메시지) |
| **Enhancement (부가)** | 식재료 DB 연동 | 바코드 스캔 또는 검색으로 제품 등록 자동화 |
| **Enhancement** | 신뢰 시스템 | 거래 후 별점·후기, 매너 온도 (당근마켓 방식) |
| **Enhancement** | 푸시 알림 | "내 동네에 파 나눔 글 올라왔어요" 알림 |
| **Enhancement** | 찜/재참여 | 자주 거래하는 이웃 팔로우, 자주 나누는 품목 즐겨찾기 |
| **Scale (확장)** | AI 수요 예측 | "이 동네에서 이 시기에 어떤 식재료 수요 높음" 예측 |
| **Scale** | 마트·새벽배송 연동 | 이마트·쿠팡 API 연동, 공동구매 주문 자동화 |
| **Scale** | 커뮤니티 레시피 | 나눈 식재료로 만든 요리 공유 피드 |

## 3단계 — MVP 스코핑 (The Lean MVP)

### MVP 범위 (Core만)

```
위치 기반 이웃 매칭 + 공동구매 방 개설/참여 + 간단 채팅 + 수동 정산 안내
```

**MVP의 목표 가설:**
> "1인 가구는 같은 동네 이웃과 식재료를 나누고 싶어하며, 간단한 매칭 기능만으로도 첫 거래가 발생한다."

### 기술 구현 현실 체크

| 항목 | 난이도 | 비고 |
|------|--------|------|
| 위치 기반 매칭 | 중 | Kakao Map API 활용 가능 |
| 방 개설/참여 | 하 | CRUD + 실시간 DB (Firebase) |
| 간단 채팅 | 중 | Firebase Realtime DB 또는 Stream Chat |
| 결제 분할 | **상** | 토스페이먼츠/포트원 연동 필요, MVP에서는 계좌이체 안내로 대체 가능 |

**현실적 MVP 일정 (2인 팀 기준):** 8~12주

> 결제 자동화는 MVP에서 제외하고 "채팅으로 계좌번호 공유" 방식으로 시작. 검증 후 결제 모듈 추가 권장.

## 4단계 — 비판적 시장 검증 (Critical Assessment)

솔직하게 말할게요. 이 아이디어는 아름다운 그림이지만, **실행의 무덤**에 빠지기 쉬운 구조입니다.

### 페인 포인트 검증: "정말 돈 낼 만큼 고통스러운가?"

**긍정 신호:**
- 통계청 기준 국내 1인 가구 비율 35% 돌파 (2024년)
- 식재료 낭비로 인한 연간 손실 체감이 크다는 설문 데이터 다수
- 당근마켓 "나눔" 카테고리에서 식재료 나눔 게시글 꾸준히 존재

**회의적 시각:**
- 실제로 사람들은 **"귀찮음"을 비용 손실보다 더 크게 느낌** → 앱 켜서 이웃 찾고 약속 잡는 게 정말 그만한 가치가 있는가?
- 식재료 단가 절감 효과가 크지 않을 수 있음 (파 한 단 2,000원 절반 = 1,000원 절감인데, 시간 비용과 맞는가?)

### 경쟁 분석

| 서비스 | 유사점 | 차별점 |
|--------|--------|--------|
| **당근마켓 나눔** | 이웃 간 물건/식재료 나눔 이미 존재 | 식재료 특화 없음, 공동구매 기능 없음 |
| **올웨이즈** | 공동구매 커머스 | 로컬/이웃 개념 없음, 배송 기반 |
| **더반찬&** | 소량 식재료 정기 배송 | B2C 구독, 커뮤니티 없음 |
| **마켓컬리 스페셜** | 소량 포장 프리미엄 | 가격대 높음, 이웃 매칭 없음 |

**핵심 차별점:** 이웃 간 실시간 소분 매칭 → 이 포지셔닝을 선점할 경쟁자는 아직 없음.

### 실패 리스크 3가지

**리스크 1: 콜드 스타트 문제 (가장 치명적)**
> 앱을 설치했는데 동네에 아무도 없으면 즉시 이탈. "닭이 먼저냐 달걀이 먼저냐" 문제.
> → 특정 아파트 단지 1~2곳에 집중 론칭하는 하이퍼 로컬 전략이 필수.

**리스크 2: 신뢰 및 안전 문제**
> 낯선 이웃과 만나는 것 자체에 거부감. 특히 여성 1인 가구는 더욱 민감.
> 식재료 신선도 분쟁, "반은 내가 가져갔는데 돈 안 줘요" 같은 소분 사기 가능성.

**리스크 3: 반복 사용 동기 부족**
> 한 번 성공하면 그 이웃과는 카톡으로 직접 연락. 앱을 거치지 않게 됨 (플랫폼 이탈).
> 락인(Lock-in) 메커니즘 없이는 DAU(Daily Active Users) 유지가 어려움.

## 5단계 — 수익화 모델링 (Monetization Strategy)

### 모델 1: 거래 수수료 (Transaction Fee)

- **구조:** 공동구매 거래 완료 시 거래액의 3~5% 수수료
- **장점:** 서비스 가치와 수익이 직결, 스케일에 따라 자연 성장
- **단점:** 초기 MAU(Monthly Active Users)가 적으면 수익 미미, 수수료 부과 순간 이탈 리스크
- **적합 단계:** PMF(Product-Market Fit, 제품-시장 적합성) 확인 후 Growth 단계

### 모델 2: 프리미엄 구독 (Freemium)

- **구조:** 무료 기본 (방 개설 월 3회) / 프리미엄 월 2,900원 (무제한 + 우선 알림 + 신뢰 배지)
- **장점:** 초기부터 수익 발생 가능, 헤비유저 세그먼트 자연 분리
- **단점:** 1,000원 아끼려는 유저에게 2,900원 구독 설득이 어려움
- **적합 단계:** MAU 5,000명 이상 확보 후 전환 실험

### 모델 3: 마트·식품 브랜드 B2B 파트너십

- **구조:** 마트 체인 또는 식품 브랜드가 "이 앱에서 우리 제품 공동구매 프로모션" 집행 → 광고비 또는 판매 수수료
- **장점:** 초기 유저 기반 없어도 파트너사 협상으로 수익 가능, 제품 DB 자동 확충
- **단점:** 영업력 필요, 파트너 의존도 높아지면 플랫폼 자율성 훼손
- **적합 단계:** 특정 지역 MAU 1,000명+ 달성 후 지역 마트 파트너십부터 시작

# 소분마켓 (SoBun Market) PRD

## 1. Project Overview

**소분마켓**은 1인 가구가 같은 동네 이웃과 대용량 식재료를 실시간으로 나눠 구매할 수 있는 하이퍼 로컬 공동구매 플랫폼이다. 식재료 낭비와 소포장 프리미엄 비용 문제를 이웃 간 연결로 해결한다.

---

## 2. Target Audience

### 핵심 페르소나: "절약형 도시 직장인 지수 (28세)"

| 항목 | 내용 |
|------|------|
| 인구통계 | 서울 마포구 원룸 거주, 연봉 3,500만 원, 비혼 |
| 행동 패턴 | 주 2회 요리 시도, 마트 방문보다 새벽배송 선호, 당근마켓 활성 유저 |
| 페인포인트 | "양배추 반통을 사도 반은 버려요. 그렇다고 소포장은 너무 비싸고." |
| 기술 수용 | 앱 설치 장벽 낮음, 신뢰 기반 서비스(당근, 에어비앤비) 사용 경험 있음 |

## 3. User Stories

- As a **1인 가구 직장인**, I want **동네 이웃이 나누려는 식재료 목록을 볼 수 있는 기능**을 원한다, so that **내가 필요한 식재료를 낭비 없이 살 수 있다**.
- As a **공동구매 개설자**, I want **원하는 인원이 모이면 자동으로 알림을 받는 기능**을 원한다, so that **직접 관리하지 않아도 구매를 진행할 수 있다**.
- As a **신규 사용자**, I want **이웃의 신뢰도와 거래 후기를 볼 수 있는 기능**을 원한다, so that **처음 만나는 사람과의 거래도 안심하고 할 수 있다**.
- As a **자주 이용하는 유저**, I want **자주 나누는 식재료 품목을 즐겨찾기하고 재알림을 받는 기능**을 원한다, so that **매번 검색하지 않아도 원하는 품목이 올라오면 바로 참여할 수 있다**.

## 6. Roadmap

### Phase 1 (MVP, 0~3개월): 가설 검증

- 서울 내 1~2개 아파트 단지에 집중 론칭 (하이퍼 로컬 전략)
- Core 기능만: 방 개설/참여 + 위치 매칭 + 채팅
- 결제는 계좌이체 안내로 대체
- **검증 목표:** "이웃과 식재료 거래 의향이 있는가?" → 방 개설 수, 거래 완료율 측정

### Phase 2 (Growth, 3~9개월): 신뢰·편의성 강화

- 자동 결제 분할 모듈 도입
- 신뢰 시스템 (별점·후기) 론칭
- 서울 5개 구로 서비스 확장
- 거래 수수료 3% 부과 시작
- **목표:** MAU 10,000명, 거래 완료율 65%

### Phase 3 (Scale-up, 9개월+): 플랫폼화

- 지역 마트/식품 브랜드 B2B 파트너십 체결
- AI 기반 수요 예측 및 선매칭 기능
- 수도권 전체 및 부산·대전 등 광역시 확장
- 커뮤니티 레시피 피드 도입으로 락인(Lock-in) 강화
- **목표:** MAU 100,000명, GMV(총 거래액) 월 5억 원

## 마지막으로, 두 가지 질문을 드립니다

**Q1.** 콜드 스타트 문제가 가장 큰 리스크인데, 처음 어느 동네(아파트 단지/오피스텔 밀집 지역)에서 집중 론칭할 계획인가요? 오프라인 커뮤니티(카카오톡 단체방, 아파트 입주민 앱) 연계 전략을 갖고 계신가요?

**Q2.** 팀 구성은 어떻게 되나요? 개발자가 있다면 네이티브 앱과 웹앱 중 어떤 방식으로 MVP를 만들 계획인지, 혼자라면 노코드(Bubble, Glide 등) 활용도 고려해보셨나요?

이 두 가지 답변을 주시면, 기술 스택과 초기 GTM(Go-To-Market, 시장 진입) 전략을 더 구체화할 수 있습니다.
